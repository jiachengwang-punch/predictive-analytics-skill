## 1. 加载数据

用 pandas 读取表格数据（CSV/Excel）。CSV 用 `pd.read_csv`，Excel 用 `pd.read_excel`。

```python
import pandas as pd
df = pd.read_csv(DATA_PATH)   # 或 pd.read_excel(DATA_PATH, sheet_name=...)
print('数据规模:', df.shape, '→', df.shape[0], '行 ×', df.shape[1], '列')
df.head()
```

**为什么先看 `.head()` 和 `.shape`**：建立对数据规模和内容的直观认识，是后续一切判断的起点。规模还决定方法选择（见下方"方法匹配约束"）。

## 2. 理解字段

为每个字段建立清晰认知，分门别类：标识类、时空类、数值特征类、类别特征类、目标变量类、结果类（事后产生）。用 `df.info()` 看类型，`df.describe()` 看数值分布，`df['col'].value_counts()` 看类别分布。

```python
df.info()                          # 类型与非空计数
df.describe().round(2)             # 数值列分布
df['某类别列'].value_counts()       # 类别取值分布
```

**这一步要回答**：数据有多大？每个字段什么含义？哪些是特征、哪些是目标、哪些是"事后结果"？时间跨度多长？这个分类直接服务于第 4 步的泄露检测。

## 3. 数据质量检查

**缺失值**：会导致模型报错或失真。
```python
print(df.isnull().sum())   # 各列缺失数
```
若有缺失：数值列可用中位数填充，类别列可用众数或单独标记为"未知"；缺失比例过高的列考虑剔除。

**重复值**：会让某些样本被重复计数。
```python
n_dup = df.duplicated().sum()
if n_dup > 0:
    df = df.drop_duplicates().reset_index(drop=True)
```

**异常值**：用 `describe()` 看是否有不合理的极值（如年龄 999、时间戳落在 1905 年）。异常占比小可过滤，占比大需深究原因。

## 4. ⭐ 数据泄露检测（按风险分级，不一刀切）

对每个候选特征，都要问一个核心问题：

> **"这个特征在『预测发生的那个时点』真的能拿到吗？还是它其实是目标变量的下游结果/同义改写？"**

如果一个特征是目标的"结果"或"测量值"，用它建模就等于**用答案预测答案**——分数会虚高到接近完美，但模型毫无实际预测价值（预测时你根本没有这个特征）。

**但泄露风险取决于数据来源，不是每种数据都需要同等强度的排查。** 问一句"有没有泄露可能"几乎零成本、应该永远做；但**投入多少精力去查，应与风险成正比**——这是"方法匹配约束"原则在质检环节的延伸。

**先做风险分级三问：**
1. 数据有时间方向吗（预测未来 / 含时间戳的事件）？
2. 特征里有没有可能是标签的下游结果、测量值或同义改写？
3. 后续建模时，会不会用**全量数据**做 target encoding、标准化、聚合等再划分训练测试？（若会，须在阶段三/四改成只在训练集上 `fit`，否则构成处理性泄露——此处先做预警。）

- **任一为"是" → 高风险**，按下方方法深查并验证。
- **全为"否" → 低风险**（干净的截面数据、语义明确的外生测量如传感器读数/物理尺寸、已策展的 benchmark、合成数据）：确认一句"各特征均为预测时点可得的外生输入，无泄露嫌疑"即可，**不必强行猎捕泄露**——那是浪费精力。

**如何识别泄露特征：**
- **逻辑同义**：某特征与标签是同一事物的不同表述。例：标签"是否超标"= "实测值 > 限值"，那么"实测值"就是泄露特征。可用一致率验证：
  ```python
  # 验证：标签是否由某特征直接算出
  recompute = (df['实测值'] > df['限值']).astype(int)
  consistency = (recompute == df['是否超标']).mean()
  print(f'一致率: {consistency*100:.1f}%')   # 若 >95%，确认泄露
  ```
- **时序泄露**：用了未来信息预测未来。例：用"本月总消费"预测"本月是否流失"，但月底才知道总消费。
- **结果类字段**：投诉量、最终评分、事后标注等，都是"事件发生后"才产生的。

**处理**：坚决剔除所有泄露特征，只保留预测时点真实可获得的特征。这一步看似简单，却是区分"专业分析"和"虚假高分"的分水岭。

**运行案例（城市噪音）**：标签"是否超标"经验证与"等效声级 > 标准限值"有 99.9% 一致率，故剔除等效声级、最大声级、噪音等级等"测量结果"，仅用交通、气象、时空等事前可知特征建模。

## 5. 时间/类型处理

把时间字符串转成 datetime，才能做时间运算和派生时间特征：
```python
df['时间列'] = pd.to_datetime(df['时间列'])
df['小时'] = df['时间列'].dt.hour
df['星期'] = df['时间列'].dt.weekday
df['是否周末'] = (df['星期'] >= 5).astype(int)
```

## 方法匹配约束（在本阶段就要心中有数）

清洗时确认的数据规模，决定后续方法选择：
- **几十到几百行**：小样本。用简单正则化线性模型（Ridge 等），严控特征数（≤ 样本数/5），用交叉验证，避免复杂模型过拟合。
- **几千到几十万行**：梯度提升树（LightGBM/XGBoost）的舒适区，通常最优。
- **几十万行以上的非结构化数据**（图像/文本）：才考虑深度学习。纯表格数据即使量大，梯度提升树通常仍胜过神经网络。

记住：方法服从约束，而非追逐时髦。
