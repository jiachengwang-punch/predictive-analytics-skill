## 1. 特征工程的三件事

对已是"一行一个样本"的结构化数据，特征工程重点是三件事：
1. **剔除泄露/标识特征**（高泄露风险数据中最关键；低风险数据主要剔除标识/常量列即可）
2. **派生有业务含义的新特征**（注入领域知识）
3. **类别变量编码**（让模型能读懂文字）

若数据是"行为日志"（一行一次行为/事件，而非一个待预测对象），则还需先**聚合**成"一行一个对象"（见第 6 节）。

## 2. 泄露/标识特征剔除

延续阶段一的泄露检测，在构造特征矩阵时明确剔除三类列：
- **标签泄露特征**：目标的测量结果/同义改写/下游效应。
- **标识/常量特征**：ID、名称、只有一个取值的列——无泛化意义。
- **另一种结果**：事后才产生的字段（如最终评分、投诉量）。

```python
leak_cols = ['实测值', '等级', ...]      # 泄露
id_cols   = ['ID', '名称', '常量列', ...] # 标识/常量
# 构造特征时排除这些列
```

## 3. 派生有业务意义的新特征

基于领域知识把原始字段转换/组合成更有信息量的特征。常见模式：
- **时间派生**：小时→时段（早高峰/日间/晚高峰/夜间）、是否周末、是否节假日。
- **分档/分箱**：连续值→有业务含义的档位。⚠️ 注意：树模型会自动找切分点，人工分箱主要帮助线性模型（见阶段五"数据驱动阈值"）。
- **比例/交互**：两列相除（转化率、密度）、相乘（交互项）。
- **二值标记**：是否属于某重点类别（如"是否敏感区"），把领域规则显式编码。

```python
# 示例：时段划分 + 二值标记
def time_period(h):
    if 0 <= h < 6:    return '夜间'
    elif 6 <= h < 9:  return '早高峰'
    elif 9 <= h < 17: return '日间'
    elif 17 <= h < 20:return '晚高峰'
    else:             return '夜晚'
df['时段'] = df['小时'].apply(time_period)
df['是否敏感类别'] = df['类别列'].isin(['A','B','C']).astype(int)
```

**为什么**：好的派生特征让模型更容易学到真实规律，也让后续特征重要性更好解读。

## 4. 类别变量编码

机器学习模型只能处理数值，类别型文字必须编码。**One-Hot 编码**最常用：把一个 k 取值的类别列展开成 k 个 0/1 列，避免给类别强加不存在的大小顺序。

```python
cat_cols = ['类别列1', '类别列2', ...]
X_encoded = pd.get_dummies(X_raw, columns=cat_cols, drop_first=False).astype(float)
```

注意：One-Hot 会增加特征维度。若某类别取值极多（高基数，如邮编），考虑目标编码或频次编码替代。

## 5. 特征与目标关系分析

这部分既是 EDA 的延续，也是论文/报告中"关键影响因素"的素材来源。详见阶段二第 6 节。核心是对比目标不同取值下各特征的差异、画分组分布图、算相关性热力图。建议在本阶段产出：

## 6. 行为日志类数据的特殊处理（聚合）

如果数据是行为日志（如"用户-商品-行为-时间"的电商日志），每行是一次行为而非一个待预测对象。此时必须先**按对象聚合**成特征表。

**关键提醒**：这类数据看似量大，但聚合后每个对象的特征信息量取决于其行为丰富度。若数据是"按行抽样"的稀疏子集（每个对象只有寥寥几次行为），聚合特征会接近常数噪声，模型学不到东西——这是常见陷阱。聚合前先检查每个对象的行为数分布（中位数若 <5，预测力会严重受限）。

```python
# 行为日志聚合为对象级特征的示例
g = data.groupby('对象ID')
feat = pd.DataFrame(index=g.size().index)
feat['行为总数'] = g.size()
feat['活跃天数'] = g['日期'].nunique()
feat['类别多样性'] = g['类别'].nunique()
# ... 各类行为计数、比例、时间偏好等
```

**运行案例对比**：城市噪音数据是"一行一记录"的结构化数据，直接建模即可；而某电商行为日志虽有 200 万行，却是按行抽样的稀疏子集（每用户中位数仅 2 次行为），导致购买预测 AUC 仅 ~0.5——这不是模型问题，而是数据稀疏性决定了任务上限。务必在聚合前诊断这一点。
