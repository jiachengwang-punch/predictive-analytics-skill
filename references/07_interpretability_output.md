## 2. SHAP 值分析

**为什么用 SHAP**：特征重要性只告诉你"谁重要"，SHAP 还告诉你"**朝哪个方向、影响多大**"。SHAP 基于博弈论的 Shapley 值，把每条预测拆解为各特征的贡献：预测值 = 基准值 + Σ(各特征贡献)。

```python
import shap
X_shap = X_test.sample(min(2000, len(X_test)), random_state=42)   # 抽样加速
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_shap)
shap.summary_plot(shap_values, X_shap, max_display=12, show=False)   # 蜂群图
```

**解读**：摘要图中每点是一个样本，横轴为 SHAP 值（对预测的影响），颜色为特征值（红高蓝低）。若某特征"高值（红）集中在右侧"，说明该特征越大越推高预测——与常识/EDA 一致则佐证模型可信。

## 3. ⭐ 诚实的模型数学表达

论文常需写出模型的数学形式，要**分类如实处理**，不能伪造不存在的公式：

**线性/逻辑回归——有解析公式**（参数化模型，训练后得到确定系数）：
- 逻辑回归：P(y=1|x) = 1 / (1 + e^−(β₀+Σβⱼxⱼ))
- 线性回归：ŷ = β₀ + Σβⱼxⱼ
可提取真实系数构造可写进报告的方程：
```python
coef = pd.Series(model.coef_[0], index=X.columns).sort_values(key=abs, ascending=False)
print('截距:', model.intercept_); print(coef.head(8))
```

**梯度提升树（LightGBM）——没有封闭公式**（这一点必须如实说明）：树集成模型是数百棵决策树相加，**不存在**线性回归那样的单一等式，强行写既不可能也无意义。应描述其**真实数学结构**：
- 加法模型：ŷ(x) = Σₘ fₘ(x)，每个 fₘ 是一棵树
- 单棵树：把特征空间划分为若干叶子区域，每个叶子输出一个常数
- 梯度提升：第 m 棵树拟合前序模型的残差，最小化带正则的目标函数

论文中应给出"加法模型通用公式 + 关键参数（树数量、叶子数等）+ 一棵样例树的结构图"，而非假等式。这正确地表述了树集成"高精度但非解析"的本质，也是选树模型的一个隐性优势（省去人工分箱，见阶段五）。
```python
import lightgbm as lgb
print('树的数量:', model.booster_.num_trees())
lgb.plot_tree(model, tree_index=0)   # 画一棵样例树展示结构
```
